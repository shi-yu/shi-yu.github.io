响应式布局meta.md

响应式布局

Meat标签

meta提供关于HTML文档的元数据。元数据不会显示在页面上，但是对于机器是可读的。它可用于浏览器（如何显示内容或重新加载页面），搜索引擎（关键词），或其他web服务。
申明编码
<meta charset="utf-8"/>


优先使用IE最新版本和Chrome
    <meta http-equiv="X-UA-Cpmpatible" content="IE=edge,chrome=1"/>

浏览器内核控制

国内浏览器很多都是双内核（webkit 和Trident）,

webkit内核高速浏览，IE内核兼容网页和旧版网站。

而添加meta标签的网站可用控制浏览器选择何种内核渲染。

<meta name="renderer" content="webkit|ie-comp|ie-stand">

国内双核浏览器默认内核模式如下：

1.搜狗高速浏览器、QQ浏览器：IE内核（兼容模式）

2.360极速浏览器、遨游浏览器：Webkit内核（极速模式）

页面关键字

<meta name="keywords" content="your tags,"/>

页面描述

<meta name="description" content="120| 150 words"/>

搜索引擎索引方式

<meata name="robots" content="index,follow"/>

all:文件将被检索，且页面上的链接可用被查询；

none:文件将不被检索，且页面上的链接不可以被查询；

index：文件将被检索；

noindex:文件将不被检索；

follow：页面上的链接可以被查询；

nofollow:页面上的链接不可以被查询。

页面重定向和刷新

<meta http-equiv="refresh" content="0;url="/>

conter内的数字代表时间（秒），既多少时间后刷新，如果加url,则会重定向到指定网页（搜索引擎能够自动检测，也很容易被引擎视作误导而受到惩罚）

网页作者信息

<meta name="author" content="author name"/>

移动设备

<meat name="viewport" content="width=device-width,initial-scale=1.0,maximum-scale=1.0,user-scalable=no"/>

WebApp全屏模式：伪装app,离线应用。

<meata name="apple-mobile-web-app-capable" content="yes"/>

隐藏状态栏/设置状态栏颜色：只有在开启WebApp全屏模式时才生效

<meta name="apple-mobile-web-app-status-bar-style"content="black-translucent"/>

content的值为default|black|black-translucent

添加到主屏后的标题

<meta name="apple-moblie-web-app-title"content="标题">
content的值为default|black|black-translucent

忽略数字自动识别为电话号码

<meta content="telephone=no"name="format-detection"/>

忽略识别邮箱

<meat content="email=no"name="format-detection"/>

css3的单位

em

倍数：相对单位，相对于父级的字号设置

rem

倍数：相对单位，相对于html根节点的字号设置

maxwidth

最大宽度：使图片自适应宽度，最大不超过父级的宽度

maxheight

最大高度：使图片自适应高度，最大不超过父级的高度

minwidth

最小宽度：使图片或内容缩小到该宽度时，不再缩小，自动换行

minheight

最小高度：使图片或内容缩小到该高度时，不再缩小，自动换行

vw

将视窗宽度分为一百份，以百分比来定义

vh

将视窗高度分为一百份，以百分比来定义

firstblog

hexo博客搭建

第一步：安装nodeJS

1.打开浏览器进入https://nodejs.org/en/

2.下载nodejs并安装

3.在终端输入node -v测试是否安装成功

第二步：安装hexo

1.在终端状态下输入 sudo npm install hero -g

2.输入密码*

3.输入hexo回车进行检测

4.创建目录mkdir blog,并进入 cd blog目录

5.输入 hexo init 初始化hexo目录

6.输入 hexo server 启动本地服务

第三步：配置github

1.在终端blog目录下配置ssh-key：输入 ssh-keygen,连续三次回车

2.再输入vim ~/.ssh/id_rsa.pub ,复制ssh-key

3.登陆github.com,注册账号和用户名，用户名是域名前缀，不要乱设，设好后登陆

4.进入后右上方有个绿色的方块 Edit profile，点击进入

5.选中左边竖目录栏SSH and GPS keys，再点击右边New SSH key，Tiele里面输入一个标题，把ssh-key复制到key里面，点击Add SSH key

6.在终端里输入git config –global user.name”shi-yu”

7.在终端里输入git config –global user.email”*“

8.新建博客域名仓库 点击右上方的加号，第一个选项，name一定要正确的输入，用户名.github.io，下方第一个和第三个复选框勾上，确定提交，然后点击右上方的Clone or download，复制地址

第四步：配置blog文件

1.用sublime text打开config.yml :在最后找到deploy,type,repe
type空格加上git，repo空格加上域名仓库的https地址，保存！

第五步：上传文件

1.在终端bolg目录下输入npm install hexo-deployer-git–save保存本地做好的文档

2.继续输入hexo g打包成html文件

3.最后输入hexo d 生成！
